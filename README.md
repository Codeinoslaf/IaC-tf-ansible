# Инфраструктура проекта: Terraform + Ansible (Yandex Cloud)
Этот репозиторий содержит код для автоматического развертывания облачной инфраструктуры в Yandex Cloud (с помощью Terraform) и её последующей конфигурации (с помощью Ansible).

## 📌 Архитектура
Terraform: Поднимает сеть (VPC), подсети, группы безопасности и две виртуальные машины на базе Ubuntu 22.04. Автоматически генерирует файл инвентаря для Ansible.

Ansible: Настраивает созданные машины с использованием ролей:

Роль common: Устанавливает базовые пакеты, создает пользователя deploy и прокидывает SSH-ключи.

Роль nginx: Устанавливает веб-сервер, деплоит конфигурацию (sites-available -> sites-enabled) и перезапускает службу через handlers.

## 🛠 Требования (Prerequisites)
Для запуска локально у вас должны быть установлены:

Terraform (версии 1.0+)

Ansible

Сгенерированная пара SSH-ключей (по умолчанию ~/.ssh/id_ed25519)

## 🚀 Быстрый старт
1. Авторизация в Yandex Cloud
Чтобы не хранить секреты в коде, мы передаем параметры доступа через переменные окружения. Перед началом работы выполните в терминале следующие три команды, подставив свои реальные данные:

```Bash
export YC_TOKEN=$(yc iam create-token)
```
```Bash
export YC_CLOUD_ID=$(yc config get cloud-id)
```
```Bash
export YC_FOLDER_ID=$(yc config get folder-id)
```
2. Развертывание инфраструктуры (Terraform)
Перейдите в директорию Terraform, инициализируйте провайдеров и запустите создание ресурсов:

```Bash
cd terraform/
terraform init
terraform apply
Нажмите yes при запросе подтверждения. После успешного выполнения Terraform автоматически создаст файл inventory.ini в папке Ansible.
```

3. Настройка серверов (Ansible)
Перейдите в директорию Ansible и запустите главный плейбук. Ansible прочитает свежие IP-адреса из инвентаря и накатит конфигурацию:

```Bash
cd ../ansible/
ansible-playbook -i inventory.ini deploy.yaml
```
📁 Структура проекта

```Plaintext
.
├── ansible/
│   ├── deploy.yaml              # Главный плейбук (точка входа)
│   ├── inventory.ini            # Файл инвентаря (ГЕНЕРИРУЕТСЯ АВТОМАТИЧЕСКИ)
│   └── roles/
│       ├── common/              # Роль: Базовая настройка ОС (пакеты, юзер deploy)
│       └── nginx/               # Роль: Установка и настройка веб-сервера
└── terraform/
    ├── main.tf                  # Вызов модулей инфраструктуры
    ├── variables.tf             # Описание глобальных переменных
    └── modules/
        ├── vpc/                 # Модуль: Сеть, подсети, Security Groups
        └── compute/             # Модуль: Виртуалки, диски + генерация inventory.ini
```
🧹 Удаление ресурсов
Если серверы больше не нужны и вы хотите остановить тарификацию в Yandex Cloud, выполните команду:

```Bash
cd terraform/
terraform destroy
```
